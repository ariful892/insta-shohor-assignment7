Some core concepts of javascript

Javascript works

Javascript normally executes the operation synchronously. But there are some functions like fetch, setTimeout, setInterval these are works asynchronously. The javascript event loop handle all the synchronous and asynchronous operation. If an synchronous statement executes it pushed in the stack and after executing it pop out from the stack. But when an asynchronous statement executes it stored in the queue and it waited there untill the response is coming from the server. In the meantime the further synchronous statement will be execting. When the response is back to the asynchronous statement that is stored in the queue then it is immediately pushed into the stack from the queue and then after executing it is pop out from the stack. This how the javascript event loop works. Though this concept is not fully clear to me but I tried to explain it to the best of my knowledge.

Local storage and Session storage both are used to store data in web browser. The data stored as a key-value pair of string. The main difference between local storage and session storage is session storage stored the data temporarily whereas local storage stored the data permanently. If a user open a tab in the browser and after finishing the task close the tab then the data stored in the session storage will be lost. But the data stored in local storage will still remain in the storage after closing the tab even after closing the browser it will remain there. And also Local storage has higher memory capacity than session storage.
